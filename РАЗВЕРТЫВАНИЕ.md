# Как опубликовать сайт-академию (пошагово)

Сайт уже собран: презентация-слайды, тест-самопроверка с подсчётом баллов и советами, библиотека «База знаний» с возможностью добавлять статьи. Осталось опубликовать.

## Шаг 1. Форк репозитория в ваш GitHub

1. Откройте оригинал: https://github.com/emin25asi-source/sks-claude-academy
2. Нажмите **Fork** (вверху справа) → создаётся копия в вашем аккаунте: `https://github.com/ВАШ-АККАУНТ/sks-claude-academy`.
3. Замените в форке файлы на эти (я подготовил обновлённый `index.html` и папку `kb/`). Проще всего — попросить Claude Code запушить эту папку в ваш форк, либо загрузить файлы через GitHub (Add file → Upload files).

## Шаг 2. Включить GitHub Pages (публикация)

1. В вашем форке: **Settings → Pages**.
2. Source: **Deploy from a branch**. Branch: **main**, папка **/(root)** → Save.
3. Через минуту сайт будет доступен по адресу:
   `https://ВАШ-АККАУНТ.github.io/sks-claude-academy/`

Сайт сразу заработает: слайды, тест-самопроверка, библиотека. Владелец/репозиторий определяются из адреса автоматически — «＋ Добавить статью» и список статей подхватятся сами.

## Шаг 3. Подключить общее хранилище для загрузки статей (Supabase)

Это нужно, чтобы загрузка файлов работала **прямо на сайте** и статьи видели все коллеги. Бесплатно, ~10–15 минут.

1. Зайдите на **supabase.com** → Sign in (можно через аккаунт GitHub) → **New project**. Имя любое, придумайте пароль БД, регион поближе (Frankfurt). Создание занимает пару минут.
2. Слева **Storage** → **New bucket** → имя **`kb`** → включите **Public bucket** → Create.
3. Слева **SQL Editor** → New query → вставьте и нажмите **Run**:
   ```sql
   create policy "kb anon read"   on storage.objects for select to anon using (bucket_id = 'kb');
   create policy "kb anon upload" on storage.objects for insert to anon with check (bucket_id = 'kb');
   ```
   (это разрешает на сайте видеть список и загружать файлы)
4. Слева **Project Settings → API**. Скопируйте:
   - **Project URL** (вид `https://xxxx.supabase.co`)
   - ключ **anon public**
5. В файле `index.html` впишите их между кавычек:
   ```js
   const SUPABASE_URL = 'СЮДА_PROJECT_URL';
   const SUPABASE_KEY = 'СЮДА_ANON_PUBLIC_КЛЮЧ';
   ```
   Сохраните/запушьте. Готово — загрузка заработает.

> `anon public` ключ безопасно публиковать: он рассчитан на работу в браузере, права ограничены политиками из шага 3.
>
> ⚠️ Важно: сейчас загружать файл может любой посетитель публичного сайта. Для внутреннего обучения это обычно нормально. Если нужно ограничить загрузку (например, паролем) — скажите, добавлю.

## Шаг 4. Как добавлять статьи в «Базу знаний»

- На сайте в разделе **Библиотека** перетащите файл в зону загрузки или нажмите **＋ Добавить статью** и выберите файл.
- Поддерживаются `.md`, `.txt`, `.pdf`, `.docx`. Файл сразу появляется плиткой у всех коллег.
- `.md`/`.txt` открываются для чтения прямо на сайте, `.pdf`/`.docx` — скачиваются по клику.
- Имя файла = заголовок плитки.
- Можно также класть «встроенные» статьи прямо в папку `kb/` репозитория — они тоже показываются в библиотеке.

## Что где лежит

| Файл / папка | Назначение |
|---|---|
| `index.html` | весь сайт (слайды, тест, библиотека) |
| `kb/` | статьи базы знаний (.md / .pdf) |
| `metodichka.pdf` | методичка курса |
| `РАЗВЕРТЫВАНИЕ.md` | этот файл |
