# Автоматическая синхронизация документов: GitHub + Obsidian + Claude Code

Пошаговая методичка для коллег. Цель — настроить общую базу знаний: документы хранятся в одном месте, обновляются у всех сами и связаны между собой.

## Что это и зачем

Связка из трёх инструментов даёт «общую базу знаний», которая обновляется автоматически:

- **Obsidian** — программа, где вы читаете и пишете документы (заметки) в виде обычных файлов на компьютере.
- **GitHub** — облачное хранилище, куда складываются все документы. Единый источник правды: у всех одинаковая свежая копия.
- **Claude Code** — ИИ-помощник в терминале. Один раз настраивает всю связку и дальше помогает вести документы.

Результат: пишете документ в Obsidian → он автоматически уезжает на GitHub → у коллег появляется через несколько минут. И наоборот.

## Почему формат .md (Markdown)

- **Совместимость и «вечность».** Простой текст откроется в любой программе и через 10 лет.
- **Git видит изменения построчно** и легко объединяет правки разных людей. Файлы Word — закрытый архив, Git их не понимает и создаёт конфликты.
- **Размер и скорость** — текст занимает килобайты.
- **Связи между документами** — ссылки `[[Документ]]` и теги работают только в текстовом формате.

## Почему нужны связи `[[ ]]`

- Контекст под рукой: из задачи один клик до договора, НПА, карточки контрагента.
- Граф связей: видно, что от чего зависит.
- Обратные ссылки: видно, кто ссылается на документ.
- Навигация вместо жёстких папок.

## Что понадобится (установить один раз)

- Аккаунт GitHub: https://github.com
- Git для Windows: https://git-scm.com
- Obsidian: https://obsidian.md
- Claude Code (если настраиваете через ИИ).

## Настройка для владельца хранилища

1. **Создать отдельный приватный репозиторий** на GitHub (New repository → имя латиницей → Private).
2. **Безопасно подключить Git к GitHub.** НЕ вписывать токен в адрес репозитория — использовать Git Credential Manager (идёт с Git for Windows, хранит доступ в зашифрованном виде). При первой синхронизации откроется окно входа — войти один раз.
3. **Связать папку Obsidian с репозиторием** (клонировать репозиторий и открыть папку как vault, либо инициализировать git в существующем сейфе — лучше через Claude Code).
4. **Установить плагин Obsidian Git** (Settings → Community plugins → отключить Restricted mode → Browse → Obsidian Git → Install → Enable).
5. **Настроить авто-коммит каждые N минут:**
   - Auto commit-and-sync interval = 10 (минут)
   - Auto pull interval = 10
   - Pull on startup = включить
6. **Выдать доступ коллегам:** Settings → Collaborators → Add people → логин коллеги.

## Настройка для коллеги (только чтение)

- Принять приглашение, склонировать репозиторий, открыть папку как vault, установить плагин Obsidian Git.
- Настройки: Auto pull interval = 60, Pull on startup = вкл., Auto commit-and-sync = 0, Auto push = 0.
- Свои заметки в общем сейфе не редактировать; для личного — отдельный сейф.

## Готовый промпт для Claude Code (владелец)

```
Помоги настроить общую базу знаний на Obsidian + GitHub с авто-синхронизацией.
Сделай по шагам и объясняй простыми словами:
1. Проверь, установлены ли Git и Obsidian; если нет — подскажи, как поставить.
2. Помоги создать отдельный приватный репозиторий на GitHub.
3. Привяжи папку моего сейфа Obsidian к репозиторию.
   ВАЖНО: не вписывай токен в адрес. Используй Git Credential Manager (вход через браузер).
4. Объясни установку плагина Obsidian Git.
5. Настрой двустороннюю синхронизацию: commit-and-sync = 10 мин, pull = 10 мин, pull on startup = вкл.
6. Объясни, как через Settings → Collaborators выдать доступ коллегам.
```

## Безопасность

- Рабочие репозитории — только Private.
- Никогда не вписывать токен в адрес и не пересылать его в переписке.
- В режиме чтения не редактировать общий сейф.
- Длинные пути в Windows: один раз выполнить `git config --global core.longpaths true`.
