## Что это за проект
Обучающий портал группы компаний **СКС** (ломбарды) по работе с ИИ.
Сейчас это **одностраничный сайт** про Claude Code для офисных сотрудников.
**Цель развития:** превратить в многостраничную академию по направлениям
группы компаний (коммерческий блок, управление персоналом, Claude Code и т.д.).

- **Автор/владелец:** Тимур Хазиев, отдел аналитики СКС.
- **Аудитория:** коллеги разных профессий и уровней подготовки.

## Где живёт и как публикуется
- **Репозиторий:** https://github.com/tikhaziev-coder/sks-claude-academy
- **Живой сайт (GitHub Pages):** https://tikhaziev-coder.github.io/sks-claude-academy/
- **Папка на Mac:** `~/sks-claude-academy`
- **Публикация:** GitHub Pages раздаёт из ветки **`main`**.
  Чтобы обновить сайт — коммит и `git push origin main`. НЕ в фиче-ветку
  (иначе сайт не обновится). После пуша Pages пересобирается ~1–2 мин.
- **Проверка сборки:** `gh api repos/tikhaziev-coder/sks-claude-academy/pages/builds/latest`
  (ждём `"status": "built"` и нужный commit sha).
- **gh / git:** `gh` установлен в `~/.local/bin` (PATH прописан в `~/.zshrc`),
  залогинен как `tikhaziev-coder`, git-auth через `gh auth setup-git`.

## Как устроен index.html
Дизайн-токены (CSS-переменные в `:root`, тёмная тема по умолчанию + светлая):
```
--bg:#130d0e  --bg2:#1b1314  --card:#231819  --card2:#2b1e1f
--ink:#f6ece9  --muted:#c3a9a4
--brand:#e31e24 (красный СКС)  --brand2:#e0a32a (золотой)  --accent:#e8833a
--grad:linear-gradient(135deg,var(--brand),var(--brand2))
--r:18px (радиус)  --maxw:1180px (ширина контента)
```
- `<html data-theme="light|dark" data-level="1">` — тема и уровень рулят CSS.
- Контент-обёртка: `.wrap` (max-width: --maxw, центр). Секции — `<section id="...">`.
- Шапка `<nav class="nav-in">`: логотип + меню + переключатель темы,
  всё **по центру**, в 3 ряда (`flex-basis:100%` у каждого блока).
- Заголовок секции — `.section-head`; карточки — `.card`.
- Адаптив: брейкпоинты 640 / 760 / 880 / 980 px.

### Видеоплеер (секция #video)
Один плеер + плейлист. Два источника:
- **YouTube** → iframe `youtube-nocookie.com/embed/<id>?autoplay=1&rel=0`

- **Google Drive** → iframe `drive.google.com/file/d/<id>/preview`.
  У Drive-видео нет родного превью → показываем заставку `logo.svg`
  через `showPoster(id,type)`, по клику → `play(id,type)`.

## Важные решения и нюансы (НЕ ломать)
- **FAQ:** секция имеет `id="faq"` (для якоря меню), а внутренний список —
  `id="faqList"` (JS: `getElementById('faqList')`). Раньше был дубль id `faq`,
  из-за чего элементы рендерились за пределами `.wrap` — не возвращать дубль.
- **Скрыто через `style="display:none"`** (по просьбе владельца):
  `.cta` (кнопки в герое), `.levelbar` («Мой уровень»), подсказка в библиотеке,
  `#libCount` (счётчик статей), `#libDrop` (дроп-зона загрузки), `#prog`
  (виджет прогресса в углу — обновлялся при каждом заходе, мешал).
- **ОТКРЫТЫЙ ВОПРОС:** селектор уровня скрыт → курс висит на уровне «Новичок»,
  блоки `data-min="2"`/`data-min="3"` (Уверенный/Продвинутый) не видны.
  Если надо показать весь контент — выставить `data-level` повыше по умолчанию.
- Локальный предпросмотр: `python3 -m http.server 8765` в фоне, открыть
  `http://localhost:8765` (НЕ через `file://` — ломает YouTube-embed).
- Видео нельзя залить через MCP Google Drive (только инлайн-контент) — большие
  файлы владелец заливает на Drive вручную и даёт share-ссылку.
