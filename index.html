С чего всё начинается

Идея за 30 секунд

Claude Code — это искусственный интеллект, который работает прямо на твоём компьютере: читает и создаёт файлы, разбирает таблицы и документы, пишет письма и отчёты, автоматизирует повторяющиеся действия. А Obsidian хранит знания и контекст, чтобы помощник всегда понимал, кто ты и над чем работаешь.

📂

Работает с файлами

Excel, Word, PDF, картинки — открывает, анализирует, создаёт новые.

⚡

Делает, а не советует

Не «вот инструкция», а сразу готовый результат у тебя на диске.

🧠

Помнит контекст

Через Obsidian знает твои проекты и не переспрашивает заново.

🔁

Автоматизирует

Рутину по расписанию: отчёты, бэкапы, напоминания.

Чат — отличное начало, особенно с «Проектами». Дальше открываются Cowork и Claude Code. Заходи с любого уровня — каждый следующий просто добавляет возможности.

Без страха · для начинающих

Начни с «Проектов» в чате

Не обязательно сразу ставить программы. В чате Claude есть «Проекты» — это твоя личная база знаний прямо в браузере. Идеальный первый шаг, чтобы попробовать и убедиться: это просто и полезно.

📁

Загрузи документы

Перетащи в Проект свои файлы: регламенты, шаблоны, инструкции, примеры. ИИ будет отвечать строго по ним, а не «из головы».

📝

Дай инструкцию

Один раз опиши: кто ты, чем занимаешься, как отвечать (тон, формат). Это «характер» помощника — он помнит это в каждом диалоге проекта.

💬

Просто спрашивай

«Найди в договоре сроки», «составь письмо по шаблону», «сделай выжимку из этих файлов». ИИ работает по твоим документам.

Создай проект

В приложении или на сайте Claude: «Проекты» → «Новый проект». Назови по теме: «Договоры», «Отчёты отдела», «Мои регламенты».

Наполни базой знаний

Загрузи документы и впиши инструкцию проекта. Чем лучше наполнишь — тем точнее и полезнее ответы.

Работай в диалоге

Задавай вопросы и задачи обычными словами. Контекст и файлы уже под рукой — повторять не нужно.

Когда захочется большего: следующий шаг — Cowork и Claude Code: они работают с файлами прямо на твоём компьютере и автоматизируют рутину. Но начать с Проектов — абсолютно нормально.

⚠️ Про персональные данные

  • Не загружай паспорта, телефоны и адреса клиентов
  • Не вставляй пароли, токены и реквизиты карт
  • Конфиденциальное — обезличивай перед загрузкой
  • Сомневаешься — лучше не грузить

Академия СКС · Claude Code

ИИ — это не страшно.
Это твой новый помощник

Не нужно быть программистом. Ты говоришь обычными словами — «сделай», «сравни», «собери» — а ИИ выполняет. Этот курс покажет, как начать без страха.

Главный страх — мимо

Ты ничего не сломаешь

  • Работаешь словами, как в переписке с коллегой — никакого кода
  • Перед важными действиями (отправка, удаление) ИИ спрашивает подтверждение
  • Файлы под контролем: есть история версий, всегда можно откатить назад
  • Начать можно с одной простой задачи на копии файла

Общая память команды

GitHub: зачем он нужен

GitHub — это облачное хранилище ваших рабочих документов. Единый источник правды: у всех всегда одинаковая свежая копия с историей изменений. Для корректной работы с Claude Code он нужен каждому.

☁️

Единое хранилище

Все документы в одном месте в облаке. Изменил у себя — у коллег появилось само. Никаких «финал_v2_итог.docx» по почте.

🕘

История версий

Каждое изменение сохраняется со временем и автором. Всегда видно, что и когда менялось, и можно откатиться назад.

🤝

Совместная работа

Команда работает над одной базой знаний. Доступ выдаётся точечно — кому читать, кому редактировать.

Почему это важно для Claude Code: помощник работает с твоими документами и контекстом. GitHub хранит их централизованно, синхронизирует между компьютерами и даёт безопасную историю — поэтому GitHub стоит завести каждому, кто хочет работать с Claude Code правильно.
Формат документов

Почему .md, а не Word или PDF

Документы базы знаний хранятся в формате .md (Markdown — простой размеченный текст). Нейросети с ним работать заметно проще, и вот почему.

✅ Markdown (.md)

  • Чистый текст — ИИ читает его целиком, без потерь и «мусора»
  • Лёгкий: килобайты, мгновенная обработка и синхронизация
  • Git видит изменения построчно — удобно сравнивать версии
  • Работают связи [[между документами]] и теги
  • Откроется в любой программе и через 10 лет

⚠️ Word (.docx) и PDF

  • Это упакованный архив со стилями — ИИ тратит силы на «распаковку»
  • PDF часто без структуры: текст «картинкой», таблицы рассыпаются
  • Git не видит изменения внутри — нормальной истории нет
  • Тяжелее и медленнее, связи и теги не работают
  • Возможны ошибки распознавания и потеря форматирования
💡 Вывод: рабочие знания держим в .md. Пришёл документ в Word или PDF — Claude Code легко перенесёт содержимое в .md. А красивый .docx для отправки наружу всегда можно собрать отдельно.

Подключение

Установка связки — по шагам

Разовая настройка. Самый простой путь — отдать её Claude Code (промпт ниже). Или вручную по шагам.

Поставь Git, Obsidian и заведи GitHub

Git — git-scm.com, Obsidian — obsidian.md, аккаунт — github.com. Claude Code подскажет, если чего-то нет.

Создай отдельный приватный репозиторий

На GitHub: New repository → имя латиницей → Private. Это твоё облачное хранилище документов.

Подключи безопасно

НЕ вписывай токен в адрес репозитория. Вход — через Git Credential Manager (откроется окно браузера, войти один раз).

Свяжи папку Obsidian с репозиторием

Папка сейфа = локальная копия репозитория. Проще сделать через Claude Code, чтобы не ошибиться в командах.

Установи плагин Obsidian Git

Settings → Community plugins → отключить Restricted mode → Browse → Obsidian Git → Install → Enable.

Включи авто-синхронизацию каждые N минут

В плагине: Auto commit-and-sync = 10, Auto pull = 10, Pull on startup = вкл. Документы будут ходить «туда-обратно» сами.

🚀 Промпт для Claude Code — настроить связку
Помоги настроить связку Git + Obsidian + Claude Code с авто-синхронизацией. По шагам, простыми словами:
1. Проверь, установлены ли Git и Obsidian; если нет — подскажи установку.
2. Помоги создать отдельный приватный репозиторий на GitHub.
3. Привяжи папку моего сейфа Obsidian к репозиторию. ВАЖНО: не вписывай токен в адрес, используй Git Credential Manager (вход через браузер).
4. Объясни установку плагина Obsidian Git.
5. Настрой двустороннюю синхронизацию: commit-and-sync = 10 мин, pull = 10 мин, pull on startup = вкл.
6. Объясни, как через Settings - Collaborators выдать доступ коллегам.
📄 Полная пошаговая инструкция — в разделе Библиотека → «Связка GitHub и Obsidian».
Самое важное

Как растить базу знаний

База знаний — это топливо помощника. Чем лучше она наполнена и ухожена, тем умнее и быстрее работает ИИ. Вот принципы, по которым её стоит пополнять.

🎯

Единый источник правды

Важное живёт в одном месте — в вальте, а не в голове, чате или десяти файлах. Изменилось — правишь один раз.

⚛️

Атомарность

Одна заметка — одна мысль или факт. Так проще найти, обновить и переиспользовать.

🔗

Связывай

Ссылки [[между заметками]] превращают список файлов в сеть знаний, где всё находится по контексту.

📥

Захват без трения

Мысль или голос в Telegram → падает в Inbox. Главное — не потерять; разберёшь и разложишь позже.

🧠

Учи память

Фиксируй правила, предпочтения и факты в памяти Claude (CLAUDE.md), чтобы не повторять одно и то же каждый раз.

Регулярная чистка

Раз в месяц — короткая ревизия: объедини дубли, обнови факты, удали устаревшее. Без ухода база превращается в свалку.

✅ Что сохранять

  • Решения и договорённости
  • Регламенты и процессы
  • Шаблоны писем, договоров, отчётов
  • Контекст проектов и клиентов
  • Твои предпочтения по работе
  • Протоколы и расшифровки встреч

❌ Что не сохранять

  • Пароли, ключи, токены
  • Персональные данные (паспорта и т.п.)
  • То, что и так лежит в рабочих системах
  • Сиюминутный мусор
  • Дубли уже записанного
  • Лишние конфиденциальные цифры

Как просить

Формулируй задачи правильно

Результат на 80% зависит от постановки задачи. Шесть проверенных приёмов.

🚀 Самый первый промпт — познакомь ИИ с собой

Прежде чем давать задачи, представься. Так помощник будет учитывать твою роль и контекст в каждом ответе. Скопируй и заполни в квадратных скобках:

Знакомство
Я начинаю работать с тобой. Запомни обо мне:
— Имя: [ФИО]. Должность: [например, юрист в компании СКС].
— Чем занимаюсь: [2–3 предложения о моих задачах].
— С какими документами работаю: [договоры, отчёты, реестры].
— Как мне удобнее получать ответы: [кратко и по делу / с таблицами / деловой тон].
— Что важно учитывать: [регламенты, сроки, особенности компании].
Дальше я буду давать тебе задачи. Учитывай это при каждом ответе и переспрашивай, если данных не хватает.
💡 В чате это можно вписать в «инструкцию проекта», в Claude Code — сохранить в память (CLAUDE.md), и тогда повторять не придётся.

Контекст + цель + формат

Кто ты, что нужно и в каком виде на выходе: таблица, письмо, одна страница.

🎯

Будь конкретным

Не «переделай отчёт», а что именно, в каком файле и каким должно стать.

🖼️

Дай образец

Пример, скрин или файл «сделай так же» — лучший способ объяснить ожидание.

🚧

Поставь рамки

«Не удаляй», «только из этой папки», «уложись в одну страницу».

🪜

Дроби на шаги

Большое — по частям, проверяя каждый шаг. Меньше ошибок, проще править.

✅

Критерий готовности

Скажи, когда считать сделано: «итоги сходятся, нет пустых строк».

❌ Слабый промпт

Переделай отчёт, чтобы было нормально.

Непонятно какой файл, что менять и каким должен быть результат.

✅ Сильный промпт

В файле report.xlsx добавь колонку «Динамика %» рядом с суммами, посчитай к прошлому месяцу, итоги внизу выдели жирным. Формат не меняй.

Файл, конкретное действие, формат и ограничение — сделает с первого раза.

Продвинутый: для крупных задач сначала попроси план (режим плана), дай 1–2 примера желаемого результата и дорабатывай итеративно по комментариям.

Первый запуск

Как начать за 5 шагов

Установи Claude Code

Десктоп-приложение для Windows/Mac (самый простой старт) либо расширение для VS Code. Войди под рабочим аккаунтом.

Поставь Obsidian и заведи базу

Создай вальт заметок, включи синхронизацию. Это «память» твоего помощника.

Открой Claude Code в рабочей папке

Например, папка с твоими таблицами и документами. Claude будет работать именно с ними.

Дай первую задачу простым языком

«Сведи эти три Excel-файла в один и посчитай итоги». Без технического жаргона — обычными словами.

Проверяй и уточняй

Смотри результат, говори что поправить. Claude дорабатывает по твоим комментариям.

Продвинутый: добавь SessionStart-хук (автозагрузка контекста), задачи в Планировщике и собственные skills под повторяющиеся процессы отдела.
